A new way to gauge success, the GNH:

Buhtan has left behind GPD in favor of GNH or Gross National Happiness.

Happiness is more ambiguous than dollars, how to we take out countries vitals?

  • Social media provides an unpreccidented and powerful tool for gauging happiness.

  • Professors Chris Danforth and Peter Dodds along with post doc Lewis Mitchell at the University of Vermont Complex Systems center have done just that.

  • The have created the Hedonometer, this monitors the happiness of the population as a whole based upon the word content of tweets.

  • The Hedonometer, courtesy of UVM Complex Systems Dept.

How can this data be visualized geographically?

  • Most tweets are sent from smartphones, and most smartphones have GPS functionality thus there is location data present in the tweets.

  • Currently than half of all tweets come from mobile users.

  • Kindly Lewis Mitchell has provided us with the raw data on the happiness of hundreds of cities around the country. We will utilize this data to visualize the distribution of happiness and compare it with the distribution of other factors. We are looking to piece together geographic correlations of factors with happiness.

import requests
import json

def make_api_string(term): 
    """Simplifies string to work with API request format.
    """
    res = term.replace(", ","+")
    return res

def geolocations(city):
    """Give me a city string and I will give you the lat and lon coordinates of it.
    """
    API_KEY = "AIzaSyByolaM8l2clFXLvBhaIRE3daYbFwK8l4Y" #our very own API key!

    #The standard JSON call function doesnt work so I made an ugly one, Luckily were only doing one thing with the api.
    url_call = "https://maps.googleapis.com/maps/api/geocode/json?address=%s&sensor=false&key=%s" % (make_api_string(city), API_KEY)
    r = requests.get(url_call) 
    results = json.loads(r.text)
    
    lat = results['results'][0]['geometry']['location']['lat'] #Why oh why so many dictionaries?!
    lon = results['results'][0]['geometry']['location']['lng']
    return (lat,lon)
